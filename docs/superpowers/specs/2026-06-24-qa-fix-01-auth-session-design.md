## Goal

Close every Auth & Session finding from the Round-1+2 audit with production-grade fixes:
real fix (no happy-path-only), structured logs/audit where relevant, tests (TDD), honest
degradation, no insecure fail-open. Two High, four Medium, six Low.

## Non-goals / out of scope (and why)

- **Boolean auth-method flags (`has_password`/`has_google`)** suggested in F-AUTH-07: a full
  redesign touches the `User` model, a migration, `UserResponse`, and the frontend. The
  *correctness bug* (avatar wiped, provider misrepresented) is fully fixed with the minimal
  change below; flags remain a future enhancement, not a deferred bug.
- **Password-reset flow** (R3 focus note): no such flow exists yet; building one is a new
  feature, not a finding fix.

### C3 — `AuthService.create_token` carries the version claim (F-AUTH-02)
`backend/app/services/auth_service.py:40`
```python
def create_token(self, user_id: str, email: str, token_version: int = 0) -> str:
    now = datetime.now(UTC)
    expire = now + timedelta(minutes=settings.jwt_expire_minutes)
    payload = {"sub": user_id, "email": email, "ver": token_version, "iat": now, "exp": expire}
    return jwt.encode(payload, settings.jwt_secret, algorithm=settings.jwt_algorithm)
```
Backward-compat: tokens minted before this change have no `ver`; readers default it to `0`
(C5), and new users start at `token_version=0`, so no forced logout on deploy.

### C13 — SQLite FK enforcement, app + tests (F-AUTH-01)
`backend/app/models/base.py` — reusable helper applied to the app engine when SQLite:
```python
def enable_sqlite_fk(async_engine) -> None:
    """Enforce FK constraints on SQLite (off by default → cascades are no-ops)."""
    from sqlalchemy import event
    @event.listens_for(async_engine.sync_engine, "connect")
    def _fk_pragma(dbapi_conn, _rec):  # noqa: ANN001
        cur = dbapi_conn.cursor()
        cur.execute("PRAGMA foreign_keys=ON")
        cur.close()

if settings.database_url.startswith("sqlite"):
    enable_sqlite_fk(engine)
```
`backend/tests/integration/conftest.py` — call `enable_sqlite_fk(eng)` inside the `engine`
fixture (right after `create_async_engine`, before `create_all`) so cascade tests exercise the
real path. The MCP-token expiry default (C12) and FK enforcement are needed for the C10/C13
regression tests to be meaningful.

## Risks / mitigations

- **Forced logout on deploy** — mitigated by `payload.get("ver", 0)` defaulting (C3/C5);
  existing tokens stay valid.
- **F-AUTH-11 inversion breaking tests/CI** — CI/test envs must resolve to a safe env. Verify
  no test instantiates `Settings(environment="staging", jwt_secret=default)`; conftest leaves
  `environment` unset → `"development"`.
- **delete_account cleanup raising** — wrap on-disk cleanup best-effort; never let artifact
  cleanup abort the user-row deletion.
