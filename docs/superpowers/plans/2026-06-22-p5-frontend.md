# P5 — Frontend: Runs, Progress, Logs & Errors — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give every index/sync trigger instant, visible progress (step N/M + %), live logs, cancel/retry, run history, and a filterable errors view — fed by one runs store that reconciles SSE (primary) with polling — and make the app's default view deterministic.

**Architecture:** Evolve `background-tasks-store` into the single source of truth keyed by `run_id`, carrying first-class progress. Triggers insert an optimistic `queued` run from the new 202 body (`{run_id, workflow_id}`) and immediately refetch pipeline-status — killing the dead air. The global pill and a rebuilt Overview panel render progress/cancel/retry from this store. A Logs/Observability screen adds Queries / Runs / Errors tabs with filters. The default view is chosen deterministically with no overview→chat flicker.

**Tech Stack:** Next.js 15 (App Router), React 19, TypeScript, Zustand, Tailwind v4 (semantic tokens only), Vitest.

**Source spec:** `…/2026-06-22-sync-and-observability-redesign-design.md` (§5.2 event fields, §7 frontend, §8 filters, §G default view). **Depends on:** P1 (202 returns `run_id`/`workflow_id`; `/tasks/active` carries `run_id`/`progress_pct`; `/runs/*` endpoints), P3 (`/logs/{p}/errors`, `/logs/{p}/runs`, `/projects/{p}/runs`).

```ts
import type { RunHistoryItem } from "./types";

export interface RunEvent {
  ts: string | null; step: string; status: string; detail: string;
  elapsed_ms: number | null; progress_pct: number | null; level: string;
}

export const runs = {
  cancel: (runId: string) =>
    request<{ cancelled: boolean; run_id: string }>(`/runs/${runId}/cancel`, { method: "POST" }),
  retry: (runId: string, forceFull = false) =>
    request<{ run_id: string; workflow_id: string; status: string }>(
      `/runs/${runId}/retry`, { method: "POST", body: JSON.stringify({ force_full: forceFull }) }),
  get: (runId: string) => request<RunHistoryItem>(`/runs/${runId}`),
  events: (runId: string, level?: string) =>
    request<RunEvent[]>(`/runs/${runId}/events${level ? `?level=${level}` : ""}`),
};
```

```ts
import { request } from "./client";
import type { ErrorLogItem, RunHistoryItem } from "./types";

export interface ErrorFilters {
  source?: string; kind?: string; failure_kind?: string; status?: string;
  date_from?: string; date_to?: string; page?: number; page_size?: number;
}

function qs(params: Record<string, string | number | undefined>): string {
  const p = new URLSearchParams();
  for (const [k, v] of Object.entries(params)) if (v !== undefined && v !== "") p.set(k, String(v));
  const s = p.toString();
  return s ? `?${s}` : "";
}

export const logs = {
  errors: (projectId: string, f: ErrorFilters = {}) =>
    request<{ items: ErrorLogItem[]; total: number; page: number; page_size: number }>(
      `/logs/${projectId}/errors${qs(f)}`),
  updateError: (projectId: string, id: string, status: string) =>
    request<{ ok: boolean }>(`/logs/${projectId}/errors/${id}`,
      { method: "PATCH", body: JSON.stringify({ status }) }),
  runs: (projectId: string, f: { kind?: string; status?: string; limit?: number } = {}) =>
    request<RunHistoryItem[]>(`/logs/${projectId}/runs${qs(f)}`),
};
```
