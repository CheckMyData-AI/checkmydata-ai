# UX Audit — 2026-07-19

- **Scope:** all (SCN-001..SCN-112)
- **Method:** static code trace (6 parallel code-inventory sweeps of `frontend/src`, evidence-backed with `file:line`)
- **Base version:** `docs/ux/scenarios.md` blob `ea840666c383a84b6edfa74c4e5d028a5d4c77a3`

## Summary

- **Totals: PASS 98 / PARTIAL 9 / FAIL 5 / BLOCKED 0** (112 scenarios)
- **Top issues (worst user damage first):**
  1. **Auth recovery is missing** — no email-verification UI (backend *enforces* it, F-PROJ-01) and no forgot/reset-password flow. A locked-out or unverified password user has no self-service path. (SCN-012, SCN-013 — FAIL)
  2. **Dishonest success / silent failure** violate vision §7 "never lie to the user": shared-dashboard "Refresh All" toasts success even when cards fail; background-task Cancel/Retry failures are swallowed; Logs Runs/Errors tabs show "no data" on fetch error. (SCN-084, SCN-105, SCN-107 — PARTIAL, major)
  3. **Dead-end flows** — dashboards can never be deleted; batch results are a "coming soon" stub; invites can only be accepted, never declined. (SCN-086, SCN-089, SCN-015 — FAIL/major)
  4. **Dropped intent** — paid pricing CTA sends logged-out users to `/login?next=/pricing` but `/login` ignores `next`, stranding them in the app. (SCN-098/SCN-110 — PARTIAL)
- **Recommended next actions (prioritized):**
  1. Build the two auth-recovery flows (SCN-012, SCN-013) — highest user harm.
  2. Fix the honesty debts as one sweep (SCN-084, SCN-105, SCN-107, SCN-067) — cheap, high-trust.
  3. Ship the three missing actions (SCN-086 delete dashboard, SCN-089 batch results, SCN-015 decline invite).
  4. Honor or drop the `next` param on login (SCN-098/110); add a clickable upgrade CTA to the 402 message (SCN-100); surface RunCard cancel/retry errors (SCN-062); add logout feedback (SCN-008).
- **Note:** all 14 findings already have spawned task chips (2026-07-19). Every PASS scenario was flipped `validated → implemented`; the 9 PARTIAL stay `validated`; the 5 FAIL and SCN-008 stay `draft`.
